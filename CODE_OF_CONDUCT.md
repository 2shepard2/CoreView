# Code of Conduct

## Our Standard

CoreView is a public technical project. Contributors are expected to communicate clearly, respectfully, and in good faith.

Examples of expected behavior:

- Be direct without being hostile.
- Assume mistakes before malice.
- Keep technical disagreement focused on the work.
- Accept feedback and correction professionally.
- Respect privacy and security boundaries.

Examples of unacceptable behavior:

- Harassment, insults, or personal attacks
- Discriminatory or exclusionary language
- Doxxing or publishing private information
- Deliberate disruption, trolling, or bad-faith escalation
- Pressuring maintainers to accept unsafe or low-quality changes

## Scope

This Code of Conduct applies to:

- GitHub issues
- Pull requests
- Discussions related to the project
- Other public spaces where people represent CoreView

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, or contributions that do not align with this Code of Conduct.

Enforcement decisions are made to protect the project, contributors, and users.

## Reporting

If you encounter behavior that violates this Code of Conduct, open a private channel with the maintainer if possible rather than escalating publicly with more damage.

## Maintainer Discretion

CoreView is a small self-hosted project. Maintainers reserve the right to enforce standards pragmatically and decisively when behavior is clearly harmful to the project or its contributors.
